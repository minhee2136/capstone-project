## Project: 최구다

Cleveland Museum of Art 소장품 기반 AI 챗봇 큐레이터 서비스.
사용자 온보딩(관심사 / 지식수준 / 관람 가능 시간) → GPT 기반 유물 추천 및 대화형 큐레이션.

앱별 역할:
- `artifacts` — CMA 소장품 데이터 + 임베딩 (구현 완료)
- `users` — 온보딩 및 사용자 프로필
- `recommendations` — 임베딩 벡터 기반 유물 추천
- `chat` — GPT 연동 대화형 큐레이션

## Commands

```bash
# Run dev server
python manage.py runserver

# Apply migrations
python manage.py migrate

# Create new migrations after model changes
python manage.py makemigrations

# Run tests
python manage.py test
python manage.py test artifacts  # single app

# Fetch artworks from Cleveland Museum of Art API
python manage.py fetch_artworks --limit 6243 --batch 100
python manage.py fetch_artworks --department "Egyptian Art"

# Generate sentence embeddings for artifacts without one
python manage.py generate_embeddings
```

## Environment Setup

Requires a `.env` file with:
```
SECRET_KEY=
DB_NAME=
DB_USER=
DB_PASSWORD=
DB_HOST=
DB_PORT=
```

Database is PostgreSQL (configured via env vars, no SQLite).

## Architecture

Django 6 REST API with four apps, only `artifacts` is actively implemented — `users`, `chat`, and `recommendations` are scaffolded but empty.

**`artifacts`** — Core data layer. The `Artifact` model mirrors Cleveland Museum of Art's Open Access API (`cleveland_id` as the external key). Each artifact stores metadata fields plus two embedding fields:
- `embedding_text`: concatenated text built at ingest time from metadata fields
- `embedding_vector`: JSON list generated by `generate_embeddings` using `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` (multilingual MiniLM)

Data pipeline: `fetch_artworks` → DB (with `embedding_vector=null`) → `generate_embeddings` → DB (vector populated).

**`users`**, **`chat`**, **`recommendations`** — Placeholder apps registered in `INSTALLED_APPS`, intended for user profiles, chat functionality, and artwork recommendations (likely using the embedding vectors).

REST Framework is configured globally with `AllowAny` permissions (no auth enforced yet).
